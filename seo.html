SEO / AEO / GEO checklist

SEO / AEO / GEO — per-article checklist (the gate)

The actionable list. Every box below traces to a spec in seo-aeo-geo-guidelines.md (read that for the why).
Most items are write-phase ticks; a few are handed down ready from research. Publish only when it passes.

Content

  • [ ] length within the target band for the type (not padded)
  • [ ] primary keyword identified · 3–5 secondary + semantic terms
  • [ ] unique value vs competitors · factually accurate + current · no scaled/thin content · duplicates canonicalised

Structure

  • [ ] one H1 w/ primary keyword near start (≤60 chars, benefit-focused)
  • [ ] 4–7 H2s · 2–3 carry a keyword variation · ≥50% phrased as questions · H1>H2>H3 (no H2→H4)
  • [ ] answer-first intro + primary keyword in first 100 words · TL;DR block above the fold
  • [ ] 40–60w liftable answer under each question H2 · FAQ = 5 Q&As · close = forward CTA, not a recap
  • [ ] no horizontal dividers in the body
  • [ ] (soft — include if it fits, not compulsory) a data/comparison table where the topic warrants one

Keywords (verify against the research keyword-set: primary · variations · secondaries · in-body)

  • [ ] primary keyword in H1 + first 100 words + 2–3 H2s + conclusion · secondaries used where mapped
  • [ ] all variations woven in naturally · no stuffing, no density-% target

Links

  • [ ] ≥3 internal links, varied descriptive anchors (<30% exact-match, never "click here"), ≤2/paragraph
  • [ ] one internal link to a product/feature page where natural (reverse-silo)
  • [ ] ≥2 external links to credible primary sources · rel policy applied (internal dofollow · citation nofollow · paid sponsored)
  • [ ] no competitor cited (auto-enforced upstream) · every link returns 200

AEO / GEO

  • [ ] direct 40–60w answer under question H2s · ≥50% H2s are questions · 4+ inline citations
  • [ ] quotable self-contained sentences (precise numbers/dates/entities) · one idea per section · ≥1 embedded video where it adds context
  • [ ] (bonus, not a gate) ≥1 original/proprietary stat

Sources & claims integrity

  • [ ] every stat → its exact source URL, returns 200, year matches, verified on the page (not memory)
  • [ ] 2–3+ distinct source domains · ≤2 citations per domain · cite-don't-promote
  • [ ] cite the most credible + freshest source available (model-decided; no fixed whitelist)
  • [ ] self-consistency: every number reconciles (no impossible math)

E-E-A-T

  • [ ] theme-native author box (named author + credentials), never in the body · datePublished + dateModified visible
  • [ ] first-hand markers in the body · citation integrity (P0) verified

Meta

  • [ ] title 50–60 chars w/ keyword · description 140–160 chars w/ keyword+value+CTA · slug has keyword · all unique + intent-matched

Readability / images

  • [ ] 15–20w sentences, 2–4-sentence paragraphs, 80%+ active · subheading every ~350w
  • [ ] images relevant + compressed · keyword-aware alt (≤125 chars) + descriptive filenames

Voice & authority

  • [ ] brand voice maintained (brand-voice.md) · written in the chosen persona's lens
  • [ ] authority honesty (position the product only where it genuinely wins) · differentiation via a real POV / illustrative use-case (no fake customer/quote/metric)

Ship

  • [ ] Overall score ≥ 80 (§7 of the full standard) · no P0 failures · avoid-ai-writing final pass run
